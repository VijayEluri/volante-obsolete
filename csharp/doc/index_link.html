Volante : Volante Developer's Guide : Link (one-to-many relation)

Link represents one-to-many relation i.e. an array of objects of a given type that are pointed to from another object (an owner persistent object). Persisted array and relation are a seperset of link functionality.

Why use link (or persisted array) instead of using a standard C# array of IPersistent objects? For efficiency.

A standard behavior for persisted object is to recursively load all referenced objects. Link and array don't do that - they just load oids of the objects and only fully load the object from the database if it's explicitly referenced.

Creating a link

To create a link use:

  • ILink<T> CreateLink<T>()
  • ILink<T> CreateLink<T>(int initialSize)

Link implements standard IList<T> interface.

Adding objects to a link

To add objects to a link:

  • void Add(T item)
  • void AddAll(T[] arr)
  • void AddAll(T[] arr, int from, int length)
  • void AddAll(ILink<T> link)
  • void Insert(int index, T item)
  • void Set(int index, T obj) - replace an object at a given index

Removing objects from a link

To remove objects from a link:

  • bool Remove(T item)
  • void RemoveAt(int index)
  • void Clear() - remove all objects

Accessing objects in a link

To access objects in a link:

  • int Length { get; set; } - get/set number of elements
  • T Get(int i)
  • bool ContainsElement(int index, T obj) - true if object at a given object is equal to obj
  • IPersistent GetRaw(int iindex) - get a raw object
  • T[] ToArray()
  • Array ToRawArray() - get an array of raw object
  • Array ToArray(Type elemType)
