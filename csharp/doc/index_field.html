Volante : Volante Developer's Guide : B-Tree field and mult-field index

B-Tree field and multi-field index are very similar to regular B-Tree index. The only difference is that the key is not any arbitrary value but must be a field of the object stored in the index.

To create field index call IFieldIndex<K, V> CreateFieldIndex<K, V>(string fieldName, IndexType indexType). fieldName is a name of a field on an object of type V. The type of that field must match type of the key K.

To create a multi-field index call IMultiFieldIndex<V> CreateFieldIndex<V>(string[] fieldNames, IndexType indexType) passing an array of names of fields.

Most of the functionality derives from IGenericIndex interface and is described in generic B-Tree documentation.

Additional field index methods

Field index provides additional methods:

  • bool Put(V obj)
  • V Set(V obj)
  • void Append(V obj)
  • V Remove(Key key)
  • V RemoveKey(K key)
  • Type IndexedClass { get; }
  • MemberInfo KeyField { get; }
Put(V obj) and Set(V obj) works like Put(K key, V obj) and Set(K key, V obj) except the key doesn't have to be provided since it can be inferred from the object itself.

If field index is of type int or long Append(V obj), in addition to peforming the job of Set(V obj), assigns an unique, auto-incremented value to the indexed field.

Remove(Key key) and RemoveKey(K key) work as in generic B-Tree index.

IndexedClass returns class of objects that can be inserted in this index.

KeyField returns info about the field key.

Multi-field index additionally provides MemberInfo[] KeyFields { get; } which returns an array with info of index fields.
