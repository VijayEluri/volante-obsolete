Volante : Introduction

Volante is a small, object-oriented, embeddable database for C#, designed for seamless integration with C# programs.

Volante is BSD-licensed (free to use even for commercial projects)

Volante is designed for similar use cases as SQLite. The difference is that Volante is written in C# and is not a SQL database. C# is object-oriented languages and objects are at its core. Storing objects in SQL databases requires converting objects to and from SQL tables (a process sometimes automated by ORMs (object-relational mappers)). This conversion process is cumbersome and slows things down. Volante understands .NET objects and can store them natively, which is faster and allows for a better integration with the host program.

Volante is not a datbase server, like MySQL. The primary use case is using Volante as a convenient way to persist and manipulate data in your .NET software (C# or any other language that targets .NET). To use Volante you add Volante.dll and it provides an API for manipulating databases.

Few things to know to understand Volante:

  • Volante stores objects
  • each stored objects gets a unique oid (object identifer)
  • oid is a 32-bit integer
  • objects can refer to other objects (via their oid)
  • there is a special root object
  • all other objects must be reachable from root object
  • you can think of Volante as a graph database: root object is a root of a graph, it points to other objects which in turn point to other objects etc.)
  • objects that are no longer reachable from root object will eventually be deleated (via database garbage collection process)
