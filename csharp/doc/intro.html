Volante : Volante Developer's Guide : Introduction to Volante

What is Volante?

Volante is a small, fast, object-oriented, NoSQL, embeddable database for C# (and other .NET languages), designed for seamless integration with .NET programs. Think of it as SQLite for .NET.

Volante is BSD-licensed (free to use even for commercial projects)

Volante is designed for similar use cases as SQLite. The difference is that Volante is written in C# and is not a SQL database but an object-oriented database.

Features overview

Volante offers a rich feature set:

  • object-oriented storage with garbage collection for removing no longer reachable objects
  • seamless integration with .NET programs
  • B-Tree indexes for quickly finding an object by a key
  • T-Tree indexes for better in-memory performance
  • R-Tree indexes for indexing spatial data
  • time series support
  • patricia trie index
  • transactions
  • schema evolution

Basics of how Volante works

Objects are at the core of .NET. Storing objects in SQL databases requires converting objects to and from SQL tables (a process sometimes automated by ORMs (object-relational mappers)). This conversion process is cumbersome and slows things down. Volante understands .NET objects and can store them natively, which is faster and allows for a better integration with your program.

Volante is not a datbase server, like MySQL. The primary use case is for using Volante as a convenient way to persist and manipulate data in your .NET programs. To use Volante, add Volante.dll to your project and it will provides an API for manipulating a database (storing, deleting, retrieving data).

Few things to know to understand Volante:

  • Volante stores .NET objects
  • each stored objects gets a unique oid (object identifer, a 32-bit integer)
  • objects can refer to other objects (via their oid)
  • there is a special root object
  • all other objects must be reachable from root object
  • you can think of Volante as a graph database: root object is a root of a graph, it points to other objects which in turn point to other objects etc.)
  • objects that are no longer reachable from root will eventually be deleated (via databasegarbage collection process)
  • indexes are used to quickly locate the data
